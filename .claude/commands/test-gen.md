Génère des tests unitaires pour le fichier ou module : $ARGUMENTS

1. Lis le code source et identifie toutes les fonctions publiques
2. Pour chaque fonction, génère des tests couvrant : cas nominal, edge cases, erreurs
3. Utilise Vitest comme framework de test
4. Place les tests dans un fichier .test.ts à côté du fichier source
5. Lance les tests et fixe ce qui casse
6. Résume la couverture : ✅ ce qui passe, ❌ ce qui échoue
